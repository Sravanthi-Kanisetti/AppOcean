Project Steps
01
🗄️
Database & Schema Creation
Create Kimball database. Create Staging schema for raw data and DW schema for the clean star schema. Done manually before any stored procedures.
Manual
02
📥
Staging Tables + Raw Load (SP1)
Create 6 staging tables with NVARCHAR types to safely accept any format. BULK INSERT all CSV files via Staging.load_data with per-table timing + TRY-CATCH error handling.
Stored Procedure
03
🔍
Data Profiling
Analyze every column using DISTINCT + COLLATE Latin1_General_CS_AS, COUNT(*)-COUNT(col) for NULLs, MAX/MIN/AVG for numerics. Find problems before touching anything.
Manual
04
🧹
Data Cleaning
Write SELECT statements to test cleaning logic first. Verify output with your own eyes. Only convert to INSERT after the result is confirmed correct.
Manual
05
⭐
DW Tables + Data Load (SP2)
Create star schema with proper data types, PK on all dims, FK on fact. Load via DW.load_data: NOCHECK FK → DELETE fact → DELETE+INSERT dims → INSERT fact → CHECK FK.
Stored Procedure
06
✅
Data Quality Validation
Compare Staging vs DW side by side using UNION ALL. Verify no data loss (row counts), NULLs handled, casing standardized, negative discounts fixed.
Manual
07
⚡
Performance Indexes
Create 5 non-clustered indexes on all FK columns (date_key, app_key, user_key, device_key, region_key) + event_type. Speed up all mart JOIN operations across 120,500 fact rows.
Optimization
08
🏪
Data Marts (6 Views)
Create 6 focused views — Marketing, Finance, Product, Regional, Device, Executive. Each joins fact + relevant dims with only business-relevant columns, including user_key for ARPU calculations.
Manual
09
📊
30+ Analytical Queries
KPI dashboards + 4-5 deep-dive queries per mart. Using LAG for MoM/QoQ/YoY trends, RANK/DENSE_RANK for top-N rankings, CTEs for multi-step comparisons, NULLIF for safe division.
Business Intelligence
10
🚀
GitHub Portfolio Packaging
Split monolithic SQL into numbered files by stage, write README with architecture flow + key decisions, push as a public portfolio project for interviews.

Advanced SQL Used
LAG()
MoM / QoQ / YoY revenue and engagement growth trends across Marketing, Finance, Product, Device & Executive marts
RANK() / DENSE_RANK()
Top-N apps per category, region, and device type — leaderboard-style ranking queries
ROW_NUMBER()
Latest-month snapshot per category/device in engagement growth tracking
SUM() OVER (PARTITION BY)
Revenue share % within a region — running window aggregation without collapsing rows
CTE (multi-level)
Chained trend logic — base metrics → LAG comparison → growth % in a single readable query
NULLIF()
Safe division — prevents divide-by-zero on every conversion, churn, and growth rate metric
COLLATE
Case-sensitive text comparison during profiling — SQL Server is case-insensitive by default
CASE (pivot-style)
Discount tier bucketing, screen size tiers, day-type classification, standardized text casing
COUNT(DISTINCT)
Unique user counts — the denominator for every ARPU calculation
CHAR(13)
Strips hidden carriage return characters left by Windows CSV exports in boolean text fields
NOCHECK CONSTRAINT
Temporarily disables FK enforcement so the fact table can be cleared and dimensions truncated safely
TRY-CATCH
Wraps both load procedures with structured error logging — error message, number, and state
